Sekce 03 — Email Parser Engine

Transformace volného textu na strukturovaná data

AI model eliminuje lidské chyby při přepisování specifikací nákladu z cizojazyčných a nestrukturovaných e-mailů[cite: 1].

Vstup: Surový e-mail zákazníka (Text)

// Unstructured email body Dobrý den, potřebujeme odvézt 3 palety z Brna (CTPark) do Hamburku. Celková váha je cca 1450 kg. Nakládka možná zítra ráno od 8:00. Prosím o rychlou kalkulaci ceny.

Uzel n8n monitoruje určenou schránku dispečinku přes bezpečný IMAP protokol.

Výstup: Validovaný JSON pro ERP

{ "origin": { "city": "Brno", "note": "CTPark" }, "destination": { "city": "Hamburg" }, "cargo": { "pallets": 3, "weightKg": 1450 }, "loadingDate": "2026-06-27T08:00:00" }

Přímý zápis do databáze dispečerského softwaru a vytvoření návrhu v systému.

Pravidla parsování v Promptu

  • Přepočet jednotek: Model automaticky konvertuje britské libry (lbs) na kilogramy a paletová místa na ložné metry (LDM).
  • Detekce chybějících dat: Pokud v e-mailu chybí hmotnost nebo rozměr, pole je označeno hodnotou null pro ruční doptání.
  • Multijazyčnost: Engine zpracovává zprávy v češtině, němčině, angličtině a polštině se stejnou přesností.
